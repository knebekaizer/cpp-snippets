bind() vs connect() on a UDP broadcast socket (Linux)
Hi all,

I'm working on a program where each instance of my program is (supposed to be) able to send broadcast UDP packets and receive same on its UDP socket. For reasons that aren't worth going into here, I'd like to be able to send broadcast packets on my UDP socket by calling connect() on the socket once, and then calling send() for each packet I want to send.... I know about sendto() but I don't want to have to call sendto() and specify the broadcast address on every call.

The problem is, calling connect() on my UDP socket seems to break things, so that while I can then send() broadcast UDP packets on my socket, I no longer recv() them. I'm seeing this problem under SUSE 10 Linux, with all instances running on the same host. What am I doing wrong?

I've included a very simple app below to demonstrate the problem... the way it should work is like a chat program; you run several instances of it and any line of text you type into one instance should appear in all the other instances.

You can see that it almost works: comment out the call to SetUDPSocketTargetToBroadcast() (on line 75), then run several instances of the program, then un-comment line 75 again, run another instance, and enter some text into that last instance. With the SetUDPSocketTargetToBroadcast() call in place, that last instance is able to send(), and all of the others are able to recv(). But what I want is, for all instances to be able to send() and recv() at all times!

-Jeremy

Code:
#include <stdlib.h>
#include <stdio.h>
#include <fcntl.h>
#include <netdb.h>
#include <unistd.h>
#include <string.h>
#include <sys/socket.h>

// My own return codes, to make the code's intent a bit more obvious
enum {
   B_ERROR = -1,
   B_NO_ERROR
};

// My return type
typedef int status_t;

// Enables or disables blocking I/O mode on the given socket
static status_t SetSocketBlockingEnabled(int sock, bool blocking)
{
   int flags = fcntl(sock, F_GETFL, 0);
   if (flags < 0) return B_ERROR;
   flags = blocking ? (flags&~O_NONBLOCK) : (flags|O_NONBLOCK);
   return (fcntl(sock, F_SETFL, flags) == 0) ? B_NO_ERROR : B_ERROR;
}

// Binds this socket to the broadcast address, so that it will receive broadcast UDP packets on (port)
static status_t BindUDPSocketToBroadcast(int sock, int port)
{
   // This allows multiple processes will be allowed to bind to the port at once.
   int one = 1;
   setsockopt(sock, SOL_SOCKET, SO_REUSEADDR, (const char *) &one, sizeof(one));
#ifdef __APPLE__
   setsockopt(sock, SOL_SOCKET, SO_REUSEPORT, (const char *) &one, sizeof(one));
#endif

   struct sockaddr_in addr;
   memset(&addr, 0, sizeof(struct sockaddr_in));
   addr.sin_family      = AF_INET;
   addr.sin_addr.s_addr = 0;   // zero means "accept data from any IP address"
   addr.sin_port        = htons(port);

   if (bind(sock, (struct sockaddr *) &addr, sizeof(addr)) == 0) return B_NO_ERROR;
                                                            else return B_ERROR;
}

// Enables/disables UDP broadcast send/receive on the given socket.
static status_t SetUDPSocketBroadcastEnabled(int sock, bool enableBroadcast)
{
   int val = (enableBroadcast ? 1 : 0);
   return (setsockopt(sock, SOL_SOCKET, SO_BROADCAST, (char *) &val, sizeof(val)) == 0) ? B_NO_ERROR : B_ERROR;
}

// Tells the socket that whenever we call send() on it, it should send the data to (broadcast) on (port)
status_t SetUDPSocketTargetToBroadcast(int sock, int port)
{
   struct sockaddr_in addr;
   memset(&addr, 0, sizeof(struct sockaddr_in));
   addr.sin_family      = AF_INET;
   addr.sin_addr.s_addr = ~0;   // ~0 means "broadcast"
   addr.sin_port        = htons(port);

   return (connect(sock, (struct sockaddr *) &addr, sizeof(addr)) == 0) ? B_NO_ERROR : B_ERROR;
}

int main(int /*argc*/, char ** /*argv*/)
{
   const int port = 9999;

   int s = socket(AF_INET, SOCK_DGRAM, 0);
   if ((s < 0)
       ||(SetSocketBlockingEnabled(s, false)     != B_NO_ERROR)
       ||(BindUDPSocketToBroadcast(s, port)      != B_NO_ERROR)
       ||(SetUDPSocketBroadcastEnabled(s, true)  != B_NO_ERROR)
       ||(SetUDPSocketTargetToBroadcast(s, port) != B_NO_ERROR)
      )
   {
      printf("Error setting up UDP broadcast Socket on port %i!\n", port);
      exit(10);
   }

   printf("UDP broadcast test running on port %i\n", port);

   char text[1000] = "";
   while(1)
   {
      fd_set readSet;
      FD_ZERO(&readSet);

      int maxfd = s;
      FD_SET(s, &readSet);

      if (STDIN_FILENO > maxfd) maxfd = STDIN_FILENO;
      FD_SET(STDIN_FILENO, &readSet);

      if (select(maxfd+1, &readSet, NULL, NULL, NULL) < 0) printf("select() failed!\n");

      if ((FD_ISSET(STDIN_FILENO, &readSet))&&(fgets(text, sizeof(text), stdin)))
      {
         // Remove the carriage return, it makes things ugly
         char * eol = strchr(text, '\n');
         if (*eol) *eol = '\0';

         int bytesSent = send(s, text, strlen(text), 0L);
         printf("Sent [%s] as a UDP packet... send() returned %i\n", text, bytesSent);
      }

      if (FD_ISSET(s, &readSet))
      {
         memset(text, 0, sizeof(text));
         int bytesRead = recv(s, text, sizeof(text), 0);
         printf("Read %i bytes of broadcast data:  [%s]\n", bytesRead, text);
      }
   }
   close(s);

   return 0;
}

First of all, instead of 0 and ~0, you'd be best off using INADDR_ANY and
INADDR_BROADCAST... Also, realize that that is merely the limited broadcast
address, and you might be better off using the actual subnet broadcast address
for your particular LAN, instead... (You can obtain that by examining your local
interfaces... Search around, I'm sure we've covered all this here before...)

Now, the main problem is that once you connect() a UDP socket, that effectively
establishes a one-to-one relationship, such that you can only send out data to that
one address, AND you can only receive data from that one address... So, anything
that arrives that is NOT from the address you connect()'d to is dropped... And, of
course, nothing that arrives will actually appear to be FROM the broadcast address
(either the limited or subnet one)... Instead, it'll be from the actual host IP that
really sent it... So, that is why you are unable to receive anything... I'm afraid
there's nothing much you can do about it, either... One possibility is to use two
separate sockets: one connect()'d to the broadcast address, which you send through;
and, one you leave unconnected, which you can receive on... You can even use
plain recv() on the unconnected UDP socket, if you like; it's exactly the same as
doing recvfrom() but passing NULL for the sender address and length pointers...
So, if you don't care to know who the sender is (which it seems like you don't, based
on your code), then it's fine to use recv() and essentially throw that info away...

Jeremy,

While Rob's suggestion of dual socket will work, but it is unnecessary. Your program can be modified to do exactly what you want with a single UDP socket.

1) You only bind() it to a sockaddr_in with the desired port number and INADDR_ANY (0).

2) You can still use select() on this UDP socket and the stdin FD.

3) You can still use recvfrom() or recv() to read from this socket.

4) To broadcast what you get from the keyboard, you MUST call sendto() using a sockaddr_in parameter with the desired port number and INADDR_BROADCAST (~0).

5) Don't use connect() for reasons explained by Rob.

For your code, you can declare "struct sockaddr_in addr;" in main() and reuse it for calling bind() and sendto(). By the way, your BindUDPSocketToBroadcast should be renamed to BindUDPSocketToAny, because INADDR_ANY (or 0) is not the broadcast address. INADDR_BROADCAST (or ~0) is the broadcast address.

I hope this helps. Have fun.
